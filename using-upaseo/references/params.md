# 参数速查表 (Parameter Quick Reference)

以下参数用于**人工覆盖** Agent 的自主判定。不传参数时 Agent 自行决策。

## 模式参数

| 参数 | 作用 | 默认行为（无参数时） |
|:---|:---|:---|
| `--quick` | 强制快速模式（跳过脑暴，单迭代） | Agent 根据任务复杂度自主判定 |
| `--full` | 强制完整仪式模式 | Agent 根据任务复杂度自主判定 |

## 网关参数

| 参数 | 作用 | 默认行为（无参数时） |
|:---|:---|:---|
| `--autopilot` | 强制所有迭代自动推进（最终 PR 仍需人工） | Agent 根据验证确定性自主判定 |
| `--gate` | 强制每个迭代都等待用户确认 | Agent 根据验证确定性自主判定 |

## 工作区参数

| 参数 | 作用 | 默认行为（无参数时） |
|:---|:---|:---|
| `--worktree` | 使用 git worktree 隔离工作区；创建后必须通过 `/upaseo-handoff --worktree` 在新 worktree cwd 下重建接收会话，后续计划/实现/验证/提交都在新会话中执行 | 关闭 |

## PR 策略参数

| 参数 | 作用 | 默认行为（无参数时） |
|:---|:---|:---|
| `--pr-per-iteration` | 每迭代单独提 PR | 关闭（统一 PR） |

## 自主判定规则速查

### 复杂度与风险判定 (micro vs quick vs full)

> **判定规则的权威定义在主文件** `using-upaseo/SKILL.md` §0.2。本速查表仅用于快速查阅，若与主文件冲突以主文件为准。

判定流程概要（完整条件与边界见主文件 §0.2）：

- **微改快速通道**：满足主文件 §0.2 列出的全部条件（≤2 文件、仅文案/注释/格式/机械修正、不改运行时逻辑/数据/API/权限/构建、Agent 有确定性、用户未要求 TDD/loop/完整仪式）。允许跳过新增失败测试、`upaseo-loop`、最小主计划和迭代设计文档；必须记录 `Micro-Change Decision` 并做最小确定性验证。无法确定风险时必须升级。
- **快速模式**：不满足微改，但满足主文件 §0.2 的全部条件（≤3 文件、不涉及架构调整或新增模块、任务意图明确）。仍必须创建最小主计划，并使用 bounded `upaseo-loop` 完成有行为风险的实现。
- **完整仪式模式**：其余情况。

### 推进判定 (auto-advance vs gate)

> **推进判定的权威定义在主文件** `using-upaseo/SKILL.md` §5.F。本速查表仅用于快速查阅。

**全部满足 → 自动推进**：
1. 验证方式为 `logs` 或 `agent-run`
2. 日志证据完整，关键事件 Trace 全部匹配预期
3. 自动化测试全部通过（若有）
4. 本迭代无架构性决策变更

**任一满足 → 必须等待用户**：
1. 验证方式包含 `browser` 或 `manual`
2. 涉及 UI/UX 视觉变更
3. 迭代中做出了设计文档之外的架构性决策
4. Agent 对验证结果存在不确定性

即使使用 `--autopilot`，最终 PR 创建、合并、ship 仍需要人工审批；autopilot 只覆盖迭代间推进，不覆盖发布合并。
